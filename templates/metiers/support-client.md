## Templates de Réponse

### Structure Standard

```
1. Salutation personnalisée
2. Reconnaissance du problème / empathie
3. Solution ou explication
4. Étapes claires si action requise
5. Offre d'aide supplémentaire
6. Signature
```

### Exemple

```
Bonjour [Prénom],

Merci de nous avoir contactés ! Je comprends que [résumé du problème] 
et je vais vous aider à résoudre ça rapidement.

Voici ce que je vous propose :
1. [Étape 1]
2. [Étape 2]
3. [Étape 3]

Si le problème persiste après ces étapes, n'hésitez pas à me le dire 
et je creuse plus en détail avec notre équipe technique.

À très vite,
[Prénom]
```

## Instructions pour le LLM

Quand je demande de l'aide support :

1. **Pour les réponses tickets** :
   - Empathie d'abord, solution ensuite
   - Étapes numérotées et claires
   - Proposer une alternative si solution principale échoue
   - Terminer par une offre d'aide
   - Ton chaleureux mais pro

2. **Pour les FAQ / Articles KB** :
   - Titre = question du user
   - Intro courte (problème + solution en 1 phrase)
   - Étapes avec screenshots si besoin
   - Section "Ça ne marche toujours pas ?"
   - Liens vers articles liés

3. **Pour les scripts d'appel** :
   - Accueil chaleureux
   - Reformuler pour confirmer le problème
   - Guider pas à pas
   - Récapituler avant de raccrocher

4. **Pour les situations difficiles** :
   - Reconnaître la frustration explicitement
   - Ne pas se justifier/défendre
   - Proposer une compensation si approprié
   - Escalader si nécessaire

5. **Format attendu** :
   - Prêt à copier/coller
   - Variables en [brackets]
   - Ton aligné avec notre voix
